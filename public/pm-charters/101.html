Design — Architecture
Browser (player) ↕ Static Frontend (game UI, timer, puzzle logic) ↕ REST API (Express) ├── GET /api/puzzle → today's puzzle ├── POST /api/score → submit nickname + score └── GET /api/leaderboard → today's rankings ↕ SQLite (puzzles, scores)
Design — Scoring Model
Base: 1pt for 4-letter words, then 1pt per letter
Obscurity: common = 1x, uncommon = 1.5x, rare = 2x
Pangram bonus: +7 points
Ranks: % of max possible score
Design — Data Model
puzzles ├── date (primary key) ├── letters (7 chars) ├── center_letter ├── word_list (JSON) └── max_score scores ├── id (auto) ├── date ├── nickname ├── score ├── words_found └── time_remaining
