Objective
Build a full-featured, browser-based project management dashboard that reads/writes charter files, tracks status, monitors deployed projects, and surfaces blockers — all from a single UI.
Success Criteria
Replaces the static charter index with a dynamic system that can filter, edit, search, and monitor all projects without touching markdown files manually.
Sign-Off
✅ Charter approved — David Kingsley · 2026-03-15
✅ In Scope
  • Filter projects by status, date, priority
  • Sort and search across all charters
  • Timeline / Gantt view of project phases
  • Expandable charter detail view
  • Edit charter status from the browser
  • Add new charter from template
  • Mark items complete with checkboxes
  • Activity feed — changes across all projects
  • Dashboard summary with health indicators
  • Ping live project URLs (up/down status)
  • Flag overdue milestones and stale blockers
🚫 Out of Scope
  • Multi-user / team collaboration
  • Kanban board / sprint management
  • Time tracking
  • External integrations (Jira, GitHub Issues)
  • Public internet access

Design — Tech Stack
LayerChoiceRationale
BackendNode.js + ExpressSame stack as existing projects
FrontendVanilla HTML/CSS/JSNo framework overhead, consistent
Data StoreFilesystem (.md files)Charters are the source of truth — no DB duplication
Markdowngray-matter + markedParse frontmatter + render markdown
HostingLocal (port 8081)Replaces current static file server
Design — Architecture
Browser (any device on LAN) ↕ http://192.168.1.124:8081 Express API Server ├── GET /api/charters → list all charters (parsed .md) ├── GET /api/charters/:id → single charter detail ├── PUT /api/charters/:id → update status/phase/items ├── POST /api/charters → create from template ├── GET /api/health → ping all live project URLs └── GET /api/activity → recent changes feed ↕ Filesystem └── charters/*.md (source of truth)
